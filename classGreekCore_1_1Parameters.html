Detailed Description

The Bridge Class for financial parameters.

Provides value semantics to the client while holding a pointer to the implementation. This allows for easy swapping of parameter types (constant vs term structure) without changing the client code (e.g., Monte Carlo engine).

Users pass this class by value. It manages the lifecycle of the inner implementation.
